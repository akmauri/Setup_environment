# Blocking Issues Requiring Human Review

This document tracks issues that require human decision before work can continue.

## Format

Each blocking issue should include:
- **Date**: When issue was raised
- **Agent**: Which agent raised it
- **Issue**: Description of the issue
- **Red Line**: Which red line is crossed
- **Impact**: What is the impact
- **Options**: Possible solutions
- **Status**: [Pending|In Review|Resolved]
- **Decision**: Final decision (when resolved)
